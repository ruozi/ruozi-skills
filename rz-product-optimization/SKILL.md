# rz-product-optimization

Post-launch optimization — data-informed experimentation flywheel.

## Scope

Iterative improvement of live products through structured experimentation. Diagnose problems, form hypotheses, run experiments, analyze results, and make decisions. Data-informed, not data-driven — human judgment matters.

## Reference

This skill extends **FL Playbook: product-optimization**. The canonical workflow and phase definitions live in:

```
../fl-playbook/product-optimization/
```

## Workflow

```
DIAGNOSE → HYPOTHESIZE → EXPERIMENT → ANALYZE → DECIDE → (repeat)
```

| Phase | Purpose | FL Playbook Reference |
|---|---|---|
| Diagnose | AARRR funnel analysis | `phases/diagnose.md` |
| Hypothesize | Form IF/FOR/THEN hypothesis | `phases/hypothesize.md` |
| Experiment | Design and run A/B test | `phases/experiment.md` |
| Analyze | Interpret results with statistical rigor | `phases/analyze.md` |
| Decide | Ship, iterate, or kill | `phases/decide.md` |

## Axioms (Summary)

Full axioms in `fl-playbook/product-optimization/principles/axioms.md`:

1. **Data-Informed** — data informs decisions, doesn't make them
2. **Hypothesis-Driven** — no experiment without a falsifiable hypothesis
3. **Single-Variable** — change one thing at a time
4. **Retention as North Star** — long-term retention > short-term conversion
5. **Learning Accumulation** — every experiment (win or loss) is permanent knowledge

## RZ Layer

Personal extensions:

- **Life metrics optimization** — apply the flywheel to personal KPIs (health, finance, productivity)
- **Habit experiments** — structured IF/FOR/THEN hypotheses for personal habit change
- **Compound learning** — log all personal experiments for pattern recognition over time
